# Bibliographic converters
### Набор библиографических инструментов для работы с ИРБИС 64
### Автор: Андрей Гасилин

#### Пакет включает в себя:
+ Модуль, предназначенный для конвертации библиографических метаданных из формата .html в табличный формат .xlsx для последующего редактирования (html_excel.py)
+ Модуль, предназначенный для конвертации готовых таблиц в формат ИРБИС 64 с последующим импортом в соответствующую библиотечную систему (excel_irbis.py)
+ Модуль, предназначенный для извлечения ненормализованных ключевых слов с добавлением в актуальную таблицу автозамены для редактирования (key_correction_IRBIS_Excel.py)
+ Модуль, предназначенный для конвертации таблицы автозамены в задания глобальной корректировки ИРБИС 64 (key_correction_Excel_IRBIS.py)
  
#### Вспомогательные модули
+ Модуль пакетной загрузки файлов (multifile_import.py)
+ Модуль генерации нормированных ключевых слов (add_keywords.py) на основе заглавия, аннотации и авторских ключевых слов
+ Модуль автоматической корректировки названий статей (correction_functions.py)
+ Модуль проверки корректности рубрик (сheck_categories.py)
+ Модуль генерации рубрик (add_categories.py) на основе ключевых слов
+ Модуль автоматического перевода (ya_trans.py)
+ Модуль обработки метаданных с помощью YandexGPT (ya_GPT.py)
+ Модуль конвертации данных из формата .xlsx в формат .json (xlsx_to_json_converter.py)

#### Авторитетные файлы
+ Таблица перевода символов в верхний регистр в именах собственных для функции автоматической корректировки (a_facet.json)
+ Нормализованный список рубрик (category_rubricator.json)
+ Нормализованный список дескрипторов с вариантами написания в виде словаря (descriptors.json)
+ Нормализованный список персоналий с вариантами написания в виде словаря (persons.json)
+ Список выходных данных избранных научных журналов (journal_list.json)
+ Актуальные списки нормализованной лексики (SNL) для разных специальностей вида 02_SNL_2026_06_22.txt

#### Проект полностью адаптирован под специальность 02, и частично адаптирован под специальности 04, 06 и 10
#### В проекте используется фреймворк FLASK, ориентированный на веб-интерфейс
